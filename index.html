Lo esencial de la tarea

El objetivo no es solo resolver un Sudoku, sino demostrar que un algoritmo de alto costo computacional puede mantenerse en C y ejecutarse desde una aplicacion web usando WebAssembly.

Backtracking en C

if (is_valid(row, column, number)) {
  grid[row][column] = number;

  if (solve_cell(row, column + 1))
    return 1;
}

Aqui ocurre el trabajo pesado: el solver prueba combinaciones de forma recursiva hasta encontrar una solucion valida. Esa es la parte ideal para ejecutar en Wasm.

Compilacion a Wasm

sudoku.c
  -> Emscripten
  -> sudoku.wasm + sudoku.js

WebAssembly no reemplaza la interfaz web: encapsula el solver en un modulo eficiente que luego puede cargarse desde JavaScript en el navegador.

Puente JS y Wasm

void set_cell(int row, int col, int value)
int solve()
int get_cell(int row, int col)

Estas funciones son la interfaz entre la web y el solver compilado: JavaScript carga el tablero, ejecuta la resolucion y luego recupera el resultado.

Inicializacion en la web

Module.onRuntimeInitialized = () => {
  // aqui JS ya puede llamar al modulo Wasm
};

Este paso es infaltable: JavaScript no debe usar `set_cell`, `solve` o `get_cell` hasta que el runtime generado por Emscripten termine de cargar.
